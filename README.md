# Subscribe App

Приложение для управления подписками с экранами онбординга, выбора тарифа и основного функционала.

## Архитектура

Проект построен по принципам Clean Architecture с использованием следующих паттернов и подходов:

- **Clean Architecture**: Четкое разделение на слои - presentation, domain, data и core
- **Управление состоянием**: Provider для управления состоянием приложения
- **Внедрение зависимостей**: get_it для управления зависимостями
- **Навигация**: Именованные маршруты с централизованным роутингом через AppRouter
- **Тестирование**: Поддержка юнит-тестов и виджет-тестов

Структура архитектуры позволяет легко масштабировать приложение и поддерживать код в чистоте. Каждый слой имеет свои четко определенные обязанности:
- [`domain`](lib/domain) - содержит бизнес-сущности и юзкейсы
- [`data`](lib/data) - реализует работу с источниками данных
- [`presentation`](lib/presentation) - отвечает за отображение и взаимодействие с пользователем
- [`core`](lib/core) - предоставляет общие утилиты и базовые классы

## Структура проекта

```
lib/
├── main.dart                 # Точка входа в приложение
├── splash_screen.dart        # Экран-заставка при запуске
├── core/                     # Общие компоненты архитектуры
│   ├── base/                 # Базовые классы
│   ├── constants/            # Константы приложения
│   ├── errors/               # Классы для обработки ошибок
│   └── utils/                # Вспомогательные утилиты
├── data/                     # Слой данных
│   ├── datasources/          # Источники данных
│   ├── models/               # DTO-классы
│   └── repositories/         # Реализации репозиториев
├── domain/                   # Бизнес-логика
│   ├── entities/             # Бизнес-сущности
│   ├── repositories/         # Абстракции репозиториев
│   └── usecases/             # Юзкейсы
├── presentation/             # Слой представления
│   └── providers/            # Провайдеры состояний
├── screens/                  # Экраны приложения
│   ├── home/                 # Главный экран
│   ├── onboarding/           # Экран онбординга
│   └── subscribe/            # Экран подписки
├── di/                       # Внедрение зависимостей
└── model/                    # Модели данных для представления
test/                         # Тесты приложения
├── domain/                   # Юнит-тесты для бизнес-логики
├── presentation/             # Тесты провайдеров
└── ui/                       # Виджет-тесты
```

## Возможные улучшения при большем времени

1. **Интеграция с платежными системами**: Реализовать подключение к реальным платежным провайдерам (Google Play Billing, App Store In-App Purchases) для обработки подписок.

2. **Локализация**: Полная реализация мультиязычности с поддержкой различных локалей через встроенные механизмы Flutter.

3. **Аутентификация пользователей**: Добавить систему регистрации и аутентификации пользователей с хранением данных в облаке.

4. **Улучшение UI/UX**: Добавить анимации, улучшить дизайн экранов, реализовать темную/светлую тему.

5. **Работа с облачными сервисами**: Интеграция с Firebase или другими облачными сервисами для синхронизации данных между устройствами.

6. **Улучшение тестового покрытия**: Добавить больше юнит-тестов для бизнес-логики и интеграционных тестов.

7. **Аналитика**: Интеграция системами аналитики для отслеживания поведения пользователей.

8. **Push-уведомления**: Реализовать систему уведомлений для напоминания о продлении подписки.

9. **Кэширование данных**: Реализовать более продвинутые стратегии кэширования для улучшения производительности.

10. **Архитектурные улучшения**: Рассмотреть возможность перехода на другие паттерны управления состоянием (Bloc, Riverpod) при необходимости.
